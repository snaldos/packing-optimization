Detailed Description

Concrete DP table implementation using a hash map for top-down DP (memoization).

Stores subproblem solutions in a sparse hash map keyed by (i, w) pairs. Used for top-down dynamic programming approaches to knapsack.

Time complexity for access/set: O(1) average (hash map) Space complexity: O(s), where s is the number of unique (i, w) states.
